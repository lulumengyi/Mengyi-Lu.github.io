Learning to Ask Good Questions: Ranking Clarification Questions using Neural Expected Value of Perfect Information

1、题目与摘要

(1) 本文针对什么任务？任务简要介绍下。

​ 本文定义了一个新任务，对于一个可能模糊的问题，从一群候选问题中找出澄清性最强的问题作为模糊问题的补充，从而得到更有价值的问题。简短来说就是通过提问对对帖子post信息补全的问题，具体来说，对于给定帖子p，和澄清性问题候选集Q，根据他们对帖子的有效性将这些问题进行排序。使用场景，比如，terry问了关于环境配置的问题，帖子的回复parker问What version of Ubuntu do you have? 当terry在写帖子的时候，系统建议了一个问题的简短列表，询问他们认为像paker这样的人可能需要提供方案的信息，从而使terry能够立即澄清他们的帖子，可能会导致更快的解决。

(2) 本文发现了什么问题？该文大体是怎么解决的？解决得如何？

询问是沟通的基础，如果机器可以自动询问问题，就可以和人类进行有效协作。本文受到 expected value of perfect information 的启发：一个好的问题是其预期答案是有用的。所以本文通过这个启发，针对这一任务设计了简单的模型，后基于Stack Exchange上爬取的数据创建了数据集，并设计了评价指标。利用StackExchange数据研究这个问题，StackExchange是一个丰富的在线资源，人们经常在其中向帖子提出澄清性问题，以便更好地为原始帖子提供帮助。本文构建了一个77K的包含帖子，问题，答案三元组的数据集，包含三个领域：askubuntu，Unix，superuser，在500个样本的数据集上对模型进行了评估，并与人类专家的判断进行对比，证明了模型在控制基线上得到重大改进。

(3) 解释下题目。题目起得如何？能概括内容并吸引人吗？

起的挺好的，小标题很吸引人。能大概概括内容

2、介绍

(1) 这个任务以往是如何解决的？作者沿着哪条路径继续研究的？为什么？

​ 关于问题生成的大多数前期工作都集中在生成阅读理解问题上，lab utov等人（2015）通过众包生成高层次的问题模板，这导致数据比本文使用的方法收集的数据要少。（2010）使用模板问题生成帮助作者编写更好的相关工作章节。Mostafazadeh等人（2016）引入一个虚拟问题生成任务，目标是生成自然问题，而不是关于图像中存在的内容，而是根据图像可以推断出的内容，与澄清问题有些类似。Penas和Hovy（2010）确定了类似于缺失信息的概念，但他们在外部知识库的帮助下对缺失信息进行补充，因此根据此本文提出了澄清问题。Artzi和Zettlemoyer（2011）使用人工生成的澄清问题驱动语义解析器，其中澄清问题旨在模拟用户查询；而本文生成澄清问题，旨在识别文本中的信息缺失。

​ 在使用社区问答论坛的任务中，关键词到问题（k2q）系统（郑等人，2011年）生成了一个可以提出问题和提炼词的列表，给定一组输入关键词，以帮助用户提出更好的问题。Figueroa和Neumann（2013）对查询的不同解释进行排名，以便在论坛上进行有效搜索。（Romeo等人，2016年）开发了一个神经网络基于工作的模型，用于在论坛上回答类似的其他问题。最近的Semeval-2017社区问题解答（CQA）（Nakov等人，2017年）任务包括一个子任务，根据评论与帖子的相关性对其进行排名。本文的任务主要区别于此任务，因为希望确定一个问题评论，它不仅与文章相关，而且还会从文章中引出有用的信息。

(2) 目前存在什么问题？为什么？你觉得可能还存在什么其他问题？为什么？

(3) 该文准备如何解决这个问题？为什么可以这样解决？你觉得该文解决这个问题的方法如何？为什么？你觉得可以如何/或更好的解决这个问题？为什么？

(4) 列出该文贡献（该文自己觉得的）

  • 基于完全信息期望值的框架，提出了一种解决排序澄清问题的神经网络模型。
  • 构建了一个源于StackExChange新的数据集，它使我们能够学习一个模型，通过查看人们提出的问题类型来提出澄清问题

4、实验

(1) 数据集及评价标准介绍

- 数据集的构建

使用StackExchange网站的数据创建数据集（post，question，answer）三元组：post是初始未编辑的post，question 是包含问题的注释，答案是在问题之后对post所做的编辑，或者作者在comments部分对问题的回复。

提取帖子：我们使用帖子历史来识别作者更新的帖子。我们使用时间戳信息来检索日志的初始未编辑版本。

提取问题：对于每个这样的文章初始版本，我们使用其评论的时间戳信息来标识对文章所做的第一个问题。我们将评论截断到问号’？’检索评论的问题部分。

提取答案：我们通过以下两种方式提取对澄清问题的答案：（a）编辑文章：作者倾向于通过编辑原始文章和添加缺少的信息来回答澄清问题。为了计算由于其他原因（如文体更新和语法更正）所做的编辑，我们只考虑长度超过四个字的编辑。作者可以对一篇文章进行多次编辑，以回答多个澄清问题。8.为了确定与给定问题评论相对应的编辑，我们选择问题后最接近的时间进行编辑。（b）对问题的回答：作者还应在评论部分对澄清问题做出后续评论。我们提取了作者在澄清问题之后的第一个结论作为问题的答案。

最后在三个标题下提取了总共77097个（post，question，answer）三元组。

- 评价标准

由于数据集中的帖子都是具有技术性质，确定有用的问题需要这方面的专家来判定，作者招募了10名具有Unix基础操作系统管理方面的经验的专家。同时向注释员提供一个帖子和一个随机列表，其中包括使用Lucene获得的十个问题候选，并要求他们选择一个“最佳”（b）问题来提问，另外标记为“有效”（v）他们认为可以在原始帖子上下文中提问的其他问题。我们要求“最好”的问题总是被标记为“有效”的问题。

在排序问题中，通常没有一个确定的阈值把得到的结果值直接判为正样本或负样本，而是采用TOP N返回的结果的Precision和recall值来衡量排序模型的性能。即认为模型返回的TOP N的结果就是模型判定的正样本，然后计算前N个位置的准确率precision@N和前N个位置的召回率recall@N。

  • precision@N 返回前N个结果的精确度，对每个样本分别计算P@N，取这些P@N 的平均值作为模型的P@N

(2) Baseline介绍

  • RANDOM ：给一个帖子，随机排列10个候选问题集。

  • BAG-NGRAM：给定一个帖子和一组10个问答候选，对帖子问题答案采用ngram表示。在所有的正、负候选三元组上训练基线，以利用（p、q）、（q、a）和（p、a）之间的交叉积特性，最大限度地减少误分类错误的影响。取ngram长度n=3。最后根据他们对正标签的预测对候选问题进行排名。

  • Community QA：SemEval 2017Community Question-Answering (CQA) 发布一个子任务，根据帖子对回复进行排序，Nandi等人（2017年）在这个任务上是第一名，利用基于字符串相似性、单词嵌入等的特征开发了一个逻辑回归模型。利用他们的模型在本文数据集进行实验。

  • Neural baselines：用LSTM对输入进行编码，

    • Neural(p, q): 帖子和问题作为输入.

    • Neural(p, a): 帖子和答案作为输入

    • Neural(p, q, a): 帖子问题答案作为输入

(3) 结果分析

  • Evaluating against expert annotations

    在非神经基线中，我们发现bag-of-ngram模型表现略好于random，但比所有其他模型都差。另一方面，Community QA表现好于神经基线（ Neural(p, q)），两者都是在不使用答案的情况下训练的。有答案的神经基线（Neural(p, q, a)和Neural(p, a)优于无答案的神经基线（Neural(p, q)），表明答案有助于选择正确的问题。更重要的是，EVPI在大多数指标上优于Neural(p, q, a)基线。两个模型使用关于真实问题和答案的相同信息，并使用相同数量的模型参数进行培训。然而，与神经基线不同的是，EVPI模型还利用备选问题和备选问题来计算其损失函数。这表明，当候选集包含与原始问题类似的问题时，对它们的效用性求和会给模型带来提升。

  • Evaluating against the original question

    bag-of-ngram模型表现类似于random，不同于对人类判断的评估。Community QA再次优于 Neural(p, q) 模型，并且与Neural(p, a)模型非常接近。与以前一样，使用答案的神经基线优于不使用答案的神经基线，EVPI模型的性能明显优于Neural(p, q, a)。

  • Excluding the original question

    神经模型优于非神经基线。然而，所有神经模型之间的差异在统计学上是不显著的。

5、结论

(1) 你觉得这篇paper创新与贡献是（不一定如作者所说）？为什么？

提出了一个新的任务，并且在实际应用中具有实用性，可以用作信息补全功能。并构建了一个问答的数据集

(2) 有没有进一步深入的价值？为什么？

有的，信息补全这个任务比较实用，本文是利用帖子下面回复的澄清问题的方式来进行对帖子进行补充信息，那应该可以利用不同方式或者模型来进行这个任务，

(3) 列出该文弱点（或者是你觉得应该是什么问题，他解决的不好，你会如何解决？）

本文只利用了评论中回复的问题，对于帖子下的评论信息应该也可以利用起来，作为对信息补充

(4) 该文对你的启发是？

启发在上网过程可以多观察多思考，是否哪些操作过程中有不便的地方，是否可以机器自动方式实现。

2、在博客根目录（注意不是archer根目录）执行以下命令：
npm i hexo-generator-json-content --save
3、在根目录_config.yml里添加配置： 
jsonContent:
  meta: false
  pages: false
  posts:
    title: true
    date: true
    path: true
    text: false
    raw: false
    content: false
    slug: false
    updated: false
    comments: false
    link: false
    permalink: false
    excerpt: false
    categories: true
    tags: true
