Learning with Noise: Enhance Distantly Supervised Relation Extraction with Dynamic Transition Matrix

1、题目与摘要

(1) 本文针对什么任务？任务简要介绍下。

​ 本文针对远程监督的关系抽取任务，远程监督是指通过将知识库与非结构化文本对齐来自动构建大量训练数据，其中主要基于以下假设：两个实体如果在知识库中存在某种关系，则包含该两个实体的非结构化句子都表示出这种关系。这个假设下会出现很多噪音，有些句子虽然同时出现了这两个实体但却没有表达这个关系，因此针对远程监督关系抽取主要问题是如何减少数据集的噪音。

(2) 本文发现了什么问题？该文大体是怎么解决的？解决得如何？

​ 针对通过远程监督构造的数据集，产生的噪音问题，作者提出使用dynamic transition对远程监督噪音数据进行建模，并curriculum learning(自主学习，课程学习)的方式训练模型，在TIMERE数据集上达到the state of art，在NYT数据集上没有使用ATT效果好。

(3) 解释下题目。题目起得如何？能概括内容并吸引人吗？

起的挺好的，突出本文重点，不遵循常规做法，不是去除掉噪音数据，而是对噪音数据建模。

2、介绍

(1) 这个任务以往是如何解决的？作者沿着哪条路径继续研究的？为什么？

之前工作主要围绕这两个方面：

  1. 通过定义规则过滤掉一些噪音数据，缺点是依赖人工定义，并且被关系种类所限制。

  2. Multi-instance learning，把训练语句分包学习，包内取平均值，或者用 attention 加权，可以中和掉包内的噪音数据。缺点是受限于 at-least-one-assumption：每个包内至少有一个正确的数据。

    前人主要思路是『去噪』，即降低噪声数据的影响。这篇文章提出用一个噪音矩阵来拟合噪音的分布，即给噪音建模，从而达到拟合真实分布的目的。

(3) 该文准备如何解决这个问题？为什么可以这样解决？你觉得该文解决这个问题的方法如何？为什么？你觉得可以如何/或更好的解决这个问题？为什么？

作者认为尽管噪音数据是不可避免的，但是用一种统一的框架对噪音数据模式进行描述是可能的。作者的出发点是，远程监督数据集中通常会有对噪音模式有用的线索。比如说，一个人的工作地点和出生地点很有可能是同一个地点，这种情形下远程监督数据集就很有可能把born-in和work-in这两个关系标签打错。

本文使用的方法是，对于每一个训练样本，对应一个动态生成的(transition matrix)。这个矩阵的作用是

  1. 对标签出错的概率进行描述
  2. 标示噪音模式

让模型先学习从输入空间到真实标签空间的映射，再用一个转移矩阵学习从真实标签空间到数据集标签空间的错误转移概率矩阵，由于对于噪音模式没有直接的监督，作者使用一种课程学习的训练方法逐渐训练模型的噪音模式，并使用迹正则（trace regularization）来控制transition matrix在训练中的行为。

  • 构建了一个源于StackExChange新的数据集，它使我们能够学习一个模型，通过查看人们提出的问题类型来提出澄清问题

3、模型

(1) 整体介绍（主要是图）

模型如图所示，使用PCNN作为sentence encoder，然后进行分类，得到句子的表示，同时，3为模型动态地产生一个transition matrix T, 用来描述噪音模式。4就是将2，3的结果相乘，得到最终结果。

换句话说，在训练阶段，使用4的输出结果，作为加噪输出和标签匹配，也就是training loss使用的是4的输出结果和训练数据的标签进行计算。而在测试阶段，使用的是2的输出结果。

训练

一开始，不考虑噪音，直接使用p计算training loss, 然后在训练过程逐渐增加o的重要性，降低p的重要性。样，在模型能够描述噪音之前， sentence encoder已经大致训练了一些。

最终的训练loss由三个部分组成，第一部分是o的loss, 第二部分是p的loss, 第三部分是迹正则。alpha, beta分别表示o,p的相对重要程度和迹正则的参数。

一开始，让alpha=1。随着训练的进行，sentence encoder 分支有了基本的预测能力。并逐渐减小beta, alpha。逐渐减小对于trace的限制，允许更小的trace来学到更多噪音。

随着训练的进行，越来越关注于学习噪音模式。

(2) 模型创新点

主要是用了动态转移矩阵来拟合噪音的分布，在训练过程中，之前的做法是，对于模型输出o, 直接计算o和label的loss, 也就是CrossEntrophy (CE) + l2 正则作为损失函数。这样做不能保证p能表示真实分布。原因在于在训练的初始阶段，对于噪音模式没有先验知识，这样T和p都不太可靠，这样会使得训练陷入局部最优。因此我们需要一种技术来保证模型能够逐步适应训练噪音数据。（利用课程学习的特点，先学习简单的知识，再逐渐学习复杂的知识），让模型能更加鲁棒，对噪音数据不受影响。

4、实验

本文分别对句子级别，Bag级别，无数据集先验知识，有先验知识进行了实验。

(1) 数据集及评价标准介绍

TIMERE

278141个句子，12种关系（所有样本都是跟时间有关系的）。时间实体粒度越细，数据越可靠。例如：

如果一句话Alphabet含有October-2-2015则很有可能描述了inception-time of（创立）这种关系。如果一句话含有Alphabet和2015，那么描述的关系可能性就会变多，例如发布财报，聘用新的CEO等等。利用这种启发式想法，本文将该数据集根据时间详细粒度分为三部分，分别代表三个不同可靠性的数据集。

分数据集的方式是：带有年月日，带有年月，带有年，以及其他时间（Wikidata中没有的三元组）的样本作为负样本。

共有负样本184579个，正样本77777（22214+2094+53469， 分别是三个可靠级别的数据）个。

validation set, test set 分别有2776， 2771个正样本，5143, 5095个负样本。

ENTITYRE 就是 NYT+Freebase

以下均使用使用PCNN做sentence encoder

sent bag开头的分别表示句子级别和Bag级别。

_mix 表示用TIMERE三个数据集一起训练

_avg表示将bag内的句子表示平均，_att表示使用attention机制

(3) 结果分析

在TIMERE数据集上效果都不错，证明使用的方法是有效的

在NYT上效果并没有太大改进，尤其是recall较低的时候，precision下降很严重。 主要原因是对于NYT并没有先验知识加入。

5、结论

(1) 你觉得这篇paper创新与贡献是（不一定如作者所说）？为什么？

从不同的角度对噪音数据进行了处理，之前都是针对【去噪】降低数据集的噪音，

(2) 有没有进一步深入的价值？为什么？

有的，对远程监督来说，噪音是个很大的问题，拟合噪音的方法是个不错的角度，这样能让模型更加鲁棒，让模型自动判断噪音数据。

(3) 列出该文弱点（或者是你觉得应该是什么问题，他解决的不好，你会如何解决？）

需要加入先验知识，人为地将训练样本分为两部分，第一部分相对可靠，第二部分相对不可靠。在NYT上效果并没有太大改进，尤其是recall较低的时候，precision下降很严重。

(4) 该文对你的启发是？

本文的这个角度很新颖，不要墨守成规，对于噪音的处理，不仅是可以去掉噪音，还可以拟合噪音，对噪音进行建模，得到噪音分布，从而让模型去拟合真实分布。
